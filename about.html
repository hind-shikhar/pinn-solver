Fundamentals

What are PINNs?

Physics-Informed Neural Networks (PINNs) are a class of deep neural networks that are trained to satisfy differential equations by embedding the physics directly in the loss function. Instead of learning from large data sets, PINNs leverage known physical laws as soft constraints — making them powerful tools for solving, discovering, and identifying partial and ordinary differential equations.

🧠

Neural Network Approximator

A deep neural network uθ(x,t) acts as a continuous, mesh-free approximation to the solution of the differential equation over the entire domain.

⚖️

Physics as Constraints

The PDE residual is evaluated at random collocation points inside the domain and penalized in the loss function, guiding the network toward the true solution.

🎯

Boundary & Initial Conditions

Known boundary and initial conditions are enforced as separate loss terms, ensuring the solution satisfies the problem's constraints at all edges of the domain.

🔍

Inverse Problem Capability

By treating physical parameters as trainable variables, PINNs can solve inverse problems — identifying unknown coefficients from sparse observations.

The Loss Function

The total training objective is a weighted combination of three terms, each encoding a different physical constraint. Minimizing this composite loss simultaneously drives the network to satisfy the PDE, boundary conditions, and any available data.

Total composite loss — weighted sum of all constraints
PDE residual loss — enforces the governing equation at collocation points
Boundary/initial condition loss — enforces known solution values at the boundary
Physics Loss

PDE Residual 𝓛physics

Computed by substituting the network output into the PDE using automatic differentiation. Evaluated at Nf randomly sampled collocation points inside the domain.

Boundary Loss

BC / IC Constraint 𝓛BC

Penalizes deviation of the predicted solution from known boundary or initial conditions at Nb boundary sample points g(x).

Data Loss (optional)

Sparse Observations 𝓛data

When labelled measurements are available, a data-fit term can be added. Essential for inverse problems where physical parameters are unknown and must be identified.

Under the Hood

How Auto-Differentiation Works

Unlike traditional numerical solvers that approximate derivatives on a fixed mesh, PINNs exploit automatic differentiation (AD) via tf.GradientTape to compute exact, symbolic-quality derivatives of the neural network output with respect to its inputs. These exact derivatives are then substituted directly into the PDE expression.

1

Forward Pass

Feed spatial/temporal coordinates (x, t) through the neural network inside a GradientTape context to produce the predicted solution û.

2

First-Order Derivatives

Call tape.gradient(u, [x, t]) to obtain ∂u/∂x and ∂u/∂t exactly, using reverse-mode AD through the network graph.

3

Higher-Order Derivatives

Nest multiple GradientTape contexts to compute ∂²u/∂x², ∂²u/∂t², or mixed partials needed for second-order PDEs.

4

PDE Residual Assembly

Combine the computed derivatives into the PDE expression — e.g. u_t − α·u_xx — and compute the mean-squared residual as the physics loss term.

5

Backpropagation

Differentiate the total loss with respect to all network weights θ and update them with Adam or L-BFGS, iterating until convergence.

// Auto-differentiation for PDE residual function computeResidual(model, x, t) { return tf.tidy(() => { const x_t = tf.variable(x); const t_t = tf.variable(t); // First-order derivatives const [u, u_x, u_t] = tf.grads( (xi, ti) => model.predict( tf.concat([xi, ti], 1) ) )([x_t, t_t]); // Second-order: u_xx via nested grad const [u_xx] = tf.grads( (xi) => tf.grads( (xi2) => model.predict( tf.concat([xi2, t_t], 1) ) )([xi])[0] )([x_t])[0]; // PDE: heat equation residual u_t - α*u_xx const alpha = tf.scalar(0.01); return u_t.sub(alpha.mul(u_xx)); }); }
Analysis

PINNs vs. Traditional Methods

How do Physics-Informed Neural Networks compare to established numerical methods like Finite Element (FEM) and Finite Difference (FDM) solvers?

Feature Traditional Methods (FEM / FDM) PINNs
Discretization ✗Requires structured mesh or grid; expensive in high dimensions ✓Mesh-free; collocation points sampled randomly in the domain
Data Needed ✓Works with governing equation alone; no training data required ✓Works with equation alone; can also incorporate sparse measurements
Solution Type ~Discrete; values only at mesh nodes, interpolation needed elsewhere ✓Continuous; evaluate solution at any point in the domain analytically
Inverse Problems ✗Requires reformulation; separate algorithms; computationally expensive ✓Natural fit; treat unknown parameters as additional trainable variables
Implementation ~Mature tooling (FEniCS, deal.II) but requires domain expertise ✓Implemented in standard ML frameworks (TF, PyTorch, JAX)
High Dimensions ✗Suffers from curse of dimensionality; mesh size grows exponentially ✓Scales better to high-dimensional problems (d > 4)
Accuracy ✓High accuracy with fine mesh; provable convergence guarantees ~Depends on network architecture; may struggle with sharp gradients
Supported Problems

Equation Types

PINNsolver supports a variety of differential equation classes, from simple one-dimensional ODEs to two-dimensional elliptic PDEs.

ODE

Ordinary Differential Equations

Functions of a single independent variable. Governed by one or more ordinary derivatives with respect to that variable.

u(t) du/dt d²u/dt²
Examples
  • Harmonic oscillator: u'' + ω²u = 0
  • Logistic growth: u' = r·u·(1 − u/K)
  • Van der Pol oscillator
  • Exponential decay
PDE · 1D

1D Partial Differential Equations

Functions of a spatial variable x and time t. Classic time-evolution problems on a 1D domain.

u(x, t) u_x u_t u_xx
Examples
  • Heat eq.: u_t = α·u_xx
  • Burgers eq.: u_t + u·u_x = ν·u_xx
  • Wave eq.: u_tt = c²·u_xx
  • Advection eq.: u_t + c·u_x = 0
PDE · 2D

2D Partial Differential Equations

Functions of two spatial variables x and y. Steady-state problems on a two-dimensional domain.

u(x, y) u_x u_y u_xx u_yy
Examples
  • Laplace eq.: u_xx + u_yy = 0
  • Poisson eq.: u_xx + u_yy = f(x,y)
  • Steady-state heat conduction
  • Electrostatic potential

Notation Guide

PINNsolver uses a compact subscript notation for partial derivatives, consistent with standard physics and mathematics literature.

Symbol Meaning Mathematical Notation Description
u_x First derivative w.r.t. x Spatial gradient in x-direction
u_t First derivative w.r.t. t Temporal rate of change
u_xx Second derivative w.r.t. x Spatial Laplacian / diffusion term
u_tt Second derivative w.r.t. t Temporal acceleration (wave eq.)
u_xt Mixed partial derivative Cross-derivative (Schwarz-symmetric)
u_yy Second derivative w.r.t. y Spatial Laplacian in y-direction
N[u] Nonlinear differential operator The full PDE expression, set to zero
u_θ Network approximation Neural network with parameters θ
